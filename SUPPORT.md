# Support

Thanks for using Quiver! Here is where to go for help.

## Questions & discussion

- **How do I…? / Is this possible?** Open a thread in
  [**GitHub Discussions**](https://github.com/achref-soua/quiver/discussions) —
  the best place for usage questions, ideas, and show-and-tell.
- **Documentation:** start with the [README](./README.md), the in-repo docs site
  (`apps/docs`, build with `mdbook build apps/docs`), and the
  ["Quiver, Explained" field guide](./docs/quiver-explained.pdf). The
  [CLI reference](./apps/docs/src/api/cli.md) and
  [configuration reference](./apps/docs/src/configuration.md) cover every command
  and environment variable; the REST API is specified in
  [`docs/api/openapi.yaml`](./docs/api/openapi.yaml).

## Bugs & feature requests

- **Found a bug?** Open an [issue](https://github.com/achref-soua/quiver/issues/new/choose)
  using the **Bug report** template (include the version, OS, and a reproduction).
- **Have an idea?** Use the **Feature request** template.

## Security

Please do **not** report security vulnerabilities in public issues. Follow
[`SECURITY.md`](./SECURITY.md) (GitHub private vulnerability reporting, or email).

## Contributing

See [`CONTRIBUTING.md`](./CONTRIBUTING.md) for the development setup, the branch
and commit conventions, and the quality gate (`just verify`).
